## 三、待完成的工作（按优先级）

### 🔴 任务 1：修复计算器↔报价联动 bug ✅ 已完成（2026-05-31）

**已修复内容**：
- `getQuoteAnalysis(quote, theoretical)` 和 `getDay2PricingScore(quote, theoretical)` 新增第二个参数，不再写死 186
- 合理区间改为相对偏移：`[theoretical+4, theoretical+34]`，拒绝线 `theoretical+74`
- `VanillaBinomialPricingTool` 新增 `onUpdateTheoretical` 回调，每次 tree 重算自动把 `Math.round(tree.vanillaPrice)` 传给父组件
- 主组件新增 `liveTheoretical` state（初始值 186），通过 `actions.updateTheoretical` 更新
- `Day2QuoteSliderPanel`、`Day2ClientResponsePanel`、`evaluateDay2`、`day2_risk_disclosure` 处理器全部改用 `liveTheoretical`
- 报价滑块下方的刻度标注（理论价/合理区间/拒绝线）也改为动态显示

**验证**：把计算器调成 σ=20%、K=23300，理论价约 350，报 380 → 合理 A；报 260 → 偏低 D。已通过 `npm run build` + `getDiagnostics`。

**问题描述**：
Day 2 的二叉树计算器和下方报价框**没有真正联动**。
- 计算器改参数后，"模型参考"数字会跟着变（比如变成 349.6）
- 但评分函数 `getQuoteAnalysis()` 永远用写死的 `day2Config.quoteRules.theoreticalPrice = 186` 来判断报价好坏
- 结果：玩家改了计算器参数，看到"模型参考 349.6"，报 260 却被判"偏贵 C"——因为评分偷偷拿 260 vs 186 比，完全无视了计算器

**根本原因**（代码位置）：
```js
// getQuoteAnalysis() 约第 1168 行
function getQuoteAnalysis(quote) {
  const theoretical = day2Config.quoteRules.theoreticalPrice; // ← 永远是 186，写死的
  ...
}
```

**修法（方案 A，推荐）**：
让评分跟着计算器的实时理论价走。需要：
1. `VanillaBinomialPricingTool` 组件把 `tree.vanillaPrice` 通过 `onUpdateQuote` 或新增 `onUpdateTheoretical` 回调传给父组件
2. 父组件把这个实时理论价传给 `getQuoteAnalysis(quote, theoreticalPrice)` 和 `getDay2PricingScore(quote, theoreticalPrice)`
3. 报价框的"数学锚点"显示实时理论价（而不是写死的 186）
4. 合理区间 `fairRange` 也改成相对理论价的百分比（如 `[theoretical + 4, theoretical + 34]`），而不是绝对数值

**验证**：改完后，把计算器调成 σ=20%、K=23300，理论价约 349，报 380 应该得 A，报 260 应该得 D（太低）。

### 🟠 任务 2：新增「中环数据台」页面（Research Terminal）✅ 已完成（2026-05-31）

**已实现内容**：
- 新增 `day2_research_terminal` stage，串接在 `day2_handbook_updated` 之后、`day2_client_arrival` 之前（`toResearchTerminal` 跳转）

- 底部「参数速查表」5 列（S₀/K/r/σ/T）方便玩家抄进计算器；每张卡有「填表提示」
- 真实数据引用 `vhsi_history.csv` / `option_chain_current.csv`，标注同组数学引擎队来源
- 已通过 `npm run build` 验证

**背景**：
用户提出了一个很好的产品想法——在玩家填写计算器之前，应该有一个"数据台"页面，让玩家**自己去查找**定价所需的关键参数（利率、波动率、现价、期限等），而不是直接给他们填好的数字。这样才能教会"sourcing the inputs"这个真实交易员技能。

**设计方案（MVP 版）**：
在 Day 2 的 `day2_tree_explainer` 阶段之前，新增一个 `day2_research_terminal` 阶段，显示一个模拟的「中环数据台」页面，包含 4 张资料卡：

| 资料卡 | 内容 | 真实数据来源 |
|--------|------|-------------|
| 📈 行情卡 | 恒指现价 21,500、VHSI 波动率指数 | `vhsi_history.csv` 中 2020-03-23 附近的真实 VHSI ≈ 60% |
| 📋 合约卡 | 真实期权链片段（行权价、到期日） | `option_chain_current.csv` 前几行 |
| 🏦 利率公告卡 | 港元无风险利率 r = 2%（HIBOR 参考） | 写死教学值，标注"参考 HIBOR" |
| 📊 指数概况卡 | 恒指股息率 q ≈ 3.5% | 写死教学值，标注"恒指年化股息率" |

玩家看完这 4 张卡，把数字记下来，然后进入计算器阶段自己填入。

**实现要点**：
- 新增 `day2_research_terminal` stage 到 `day2Config.stages`
- 新增 `Day2ResearchTerminalPanel` 组件，4 张卡片布局
- 在 stage 流程里插入到 `day2_handbook_updated` 之后、`day2_tree_explainer` 之前
- MVP 阶段不需要校验玩家填的对不对，只是展示数据让玩家参考
- 卡片风格参考现有的 `TerminalCard` + `TerminalHeader` 组件，保持视觉一致

**注意**：这个任务依赖任务 1 完成（计算器联动修好后，数据台才有意义）。
