## 0. 一句话结论

这是一个**结构清晰、教学节奏好、合规意识强**的游戏化期权教学项目，四天的能力曲线（普通期权 → 二叉树定价 → 障碍期权 → 牛熊证）设计得相当专业。核心二叉树定价数学**正确**。

但存在 **1 个必须修复的金融结构错误（牛熊证收回价与行权价的大小关系颠倒）**，以及 **几处跨关卡的数值不一致**（同一份普通期权在不同天里价格不同）。修复这些之后，项目的金融严谨度会显著提升，足以经受懂行同学/老师的提问。

## 1. 金融知识准确性审查（重点）

### 🔴 严重 1：熊证（Bear CBBC）的「行权价 vs 收回价」大小关系颠倒 — ✅ 已修复（2026-05-31）

**位置**：`day4Config.market`（约第 1068–1075 行）

```js
market: {
  underlying: "恒生指数",
  spot: 21500,
  strike: 21200,          // ❌ 行权价低于现价，也低于收回价
  cbbcCallPrice: 22000,   // 收回价
  cbbcEntryCost: 80,
  vanillaPremium: 150,
  maturity: "1个月",
  path: [21500, 21780, 22050, 21600, 21100, 20750],
}
```

**问题**：港交所牛熊证的真实结构有严格的价格排序：

| 类型 | 正确排序 | MCE 触发 |
|------|----------|----------|
| 牛证 Bull | 行权价 ≤ 收回价 ≤ 现价 | 标的**跌**到收回价 |
| 熊证 Bear | 现价 ≤ 收回价 ≤ **行权价** | 标的**涨**到收回价 |

熊证要求 **收回价 ≤ 行权价**。当前配置是 `行权价 21200 < 收回价 22000`，**正好反了**，而且行权价还低于现价（21500），这在真实市场上不是一张合法的熊证。

> 注：这是手册里教的对称关系「牛证危险线在下方、熊证危险线在上方」的延伸——收回价的位置对了（22000 在现价上方 ✓），但行权价的位置错了。

**实际采用的修复（解耦方案，已落地）**：

发现 `market.strike` 在 Day 4 被**两个产品共用**——既是熊证行权价，又是「普通 Put 比较产品」的行权价（图表标注「普通 Put 行权价」，期权费 150）。若简单把 `strike` 改成 22000，熊证结构修好了，但普通 Put（现价 21500、行权价 22000）会变成内在价值 500 点、却只卖 150 点，**制造一个新的套利错误**。因此采用解耦：

```js
market: {
  underlying: "恒生指数",
  spot: 21500,
  strike: 21200,          // ✅ 普通 Put 比较产品行权价（价外 Put，保持合理）
  cbbcStrike: 22000,      // ✅ 新增：熊证自己的行权价（N 类熊证 = 收回价，无残值）
  cbbcCallPrice: 22000,   // 收回价（现价 21500 ≤ 收回价 22000 ≤ 熊证行权价 22000 ✓）
  cbbcEntryCost: 80,
  vanillaPremium: 150,
  maturity: "1个月",
  path: [21500, 21780, 22050, 21600, 21100, 20750],
}
```

排序验证：现价 21500 ≤ 收回价 22000 ≤ 熊证行权价 22000 ✓（N 类熊证）；普通 Put 行权价 21200 < 现价 21500 → 价外 Put，期权费 150 合理 ✓。

**连带修改**：`getDay4MarketResult()`（约第 1306–1324 行）中熊证未敲出收益由 `Math.max(market.strike - finalPrice, 0)` 改为 `Math.max(market.cbbcStrike - finalPrice, 0)`；普通 Put 仍用 `market.strike`。本条路径中途触及 22050 ≥ 22000 → 触发 MCE，熊证结算仍为 `-cbbcEntryCost`，剧情不变。已通过 `npm run build` 验证。✅

### 🟠 中等 2：牛熊证的「成本 / 收益」尺度内部不自洽 — ✅ 已修复（2026-05-31）

**位置**：`getDay4MarketResult()`（第 1306–1324 行）+ `cbbcEntryCost: 80`

**问题**：代码把熊证「未敲出」的收益直接复用了 Put 的 payoff 公式 `max(strike - finalPrice, 0)`，却把成本设成 80 点。如果行权价按严重 1 改为 22000，那么这张熊证在入场时相对现价（21500）就已经有约 500 点的「内在价值」（strike − spot = 22000 − 21500），但成本只标 80 点——这相当于花 80 点买一个立刻就值 500 点的东西，**接近无风险套利**，金融上不自洽。

根因：牛熊证不是用「期权费」定价的，它更像一个**带杠杆的线性跟踪工具**（价格 ≈ |行权价 − 现价| × 换股比率 + 财务费用），而代码用「期权 payoff − 期权费」的框架去套它。

**两个修法，二选一**：

- **方案 A（最小改动，推荐用于教学）**：保留现有公式，但在 UI 和手册里**明确标注**「成本/收益为示意性杠杆数值，非真实换股比率定价」，并把 `cbbcEntryCost` 与未敲出收益放到可比尺度。例如把熊证未敲出收益改为「相对入场的变动」而非绝对内在价值：
  ```js
  // 未敲出：以入场参考价为基准的杠杆盈亏（示意）
  const bearReferenceValue = Math.max(market.strike - market.spot, 0); // 入场内在 ≈ 500
  const bearFinalValue = Math.max(market.strike - finalPrice, 0);       // 结算内在
  const bearCbbcPnl = mceTriggered
    ? -market.cbbcEntryCost
    : (bearFinalValue - bearReferenceValue) /* 杠杆变动 */ - 0;
  ```
  这样「方向对但没敲出」赚的是价格变动部分，而不是凭空冒出 500 点。

- **方案 B（更直观）**：把熊证彻底建模成「带杠杆的看跌敞口」：设杠杆倍数 `leverage`（如 5×），成本 = 投入资金，未敲出收益 = `leverage × (spot − finalPrice) / spot × 投入资金`，敲出 = 损失全部投入。这能真实体现「牛熊证下跌反应更快」的卖点。

> 业务经理视角：这条不影响剧情和评分逻辑，但**懂行的同学一眼会看出 80 点成本对不上 500 点价值**。建议至少做方案 A 的「示意性」标注 + 尺度对齐。

> **实际采用：方案 B**。新增 `cbbcLeverage: 7`，熊证未敲出盈亏 = `cbbcEntryCost × 杠杆 × 标的下跌比例`（以 `-cbbcEntryCost` 为下限），触发 MCE 损失全部投入。同时在 UI（熊证成本卡片标「约 7× 杠杆，示意」）和手册（新增「价格与杠杆（教学简化）」一节）做了「示意性数值」披露。已 `npm run build` 验证通过。

### 🟠 中等 3：同一份普通看涨期权，在不同天里价格不一致 — ✅ 已修复（2026-05-31）

**问题**：三处描述的是**完全相同**的合约（恒指、现价 21500、行权价 22000、1 个月），价格却不同：

| 位置 | 字段 | 普通 Call 价格 |
|------|------|----------------|
| `day1Config.market.premium` | 第 283 行 | ~~150~~ → **186** ✅ |
| `day2Config.quoteRules.theoreticalPrice` / `tree.theoreticalPrice` | 第 458 / 550 行 | **186** |
| `day3Config.market.vanillaPremium` | 第 817 行 | ~~150~~ → **186** ✅ |

我手算验证过：Day 2 的二叉树参数（up +2.65%、down −2.58%、r=2%、3 步、1 个月）算出的理论价 ≈ **186.2**，所以 Day 2 的 186 是对的。但 Day 1/Day 3 用 150 描述同一合约，会让连续游玩的玩家产生「昨天还 150，今天怎么变 186」的困惑。

**修改建议（统一参数，二选一）**：

- **方案 A（推荐）**：把 Day 1、Day 3 的普通 Call 价格统一改为 **186**，与 Day 2 的模型价格一致；Day 3 的障碍期权（down-out call）相应改为约 **110–120**（仍明显低于普通 Call，体现「便宜来自敲出风险」）。
- **方案 B**：调整 Day 2 二叉树参数（如降低 sigma），让理论价落到 150，再三处统一为 150。

> 统一后，跨关卡叙事会更可信：Day 2 学到「这张 Call 值 186」，Day 3 就能顺势说「同样这张 186 的 Call 太贵，下跌敲出版只要 ~115」。

### 🟡 轻微 4：障碍/CBBC 的「障碍监控」是离散的（只在树节点/路径点检查）

**位置**：
- 障碍树 `buildBarrierBinomialToolTree`（第 ~3772 行）：`knocked: price <= barrier`
- Day 3 结算 `getDay3MarketResult`（第 1286 行附近）：`market.path.findIndex(price => price <= barrier)`
- Day 4 MCE（第 1308 行）：`market.path.findIndex(price => price >= cbbcCallPrice)`

**问题**：真实障碍/牛熊证是**连续监控**的（盘中任意时刻触线即敲出/收回）。代码只在离散的路径点/树节点上检查，会**低估敲出概率**，从而**高估障碍期权价值**。这是标准的教学简化，本身可接受。

**建议**：不改公式，但在手册「障碍期权」与「牛熊证」页里加一句披露：「本模拟采用离散观察点，真实产品为连续监控，敲出更容易发生。」这正好呼应你们已有的「诚实披露」风格，也是一个加分的专业细节。

### 🟡 轻微 5：定价模型未含股息收益率 q（与同学的数学引擎不一致）

**位置**：`buildVanillaBinomialToolTree` / `buildBarrierBinomialToolTree`（第 3667 / 3754 行起），`growth = Math.exp(rate * dt)`。

**问题**：二叉树的风险中性增长率只用了无风险利率 r，没有扣除股息率 q。而你同学的 `central-trader-en/analysis_report_step1-7.md` 里**专门强调**恒指股息率 ≈ 3.5%，不含 q 会系统性高估 Call 约 3%。两个小组对外展示时，口径不一致会被注意到。

**建议**：
- 教学上 1 个月期限影响很小，可保留简化；但建议在手册里加一句「教学简化：暂不考虑股息率 q，真实恒指定价需扣除约 3.5% 年化股息」。
- 若想与数学引擎口径统一，把增长率改为 `Math.exp((rate - q) * dt)` 并引入 `q` 参数（默认 0.035）。

### 🟡 轻微 6：定价台 P&L 是「单路径、无对冲」的裸头寸盈亏

**位置**：`getDay2MarketResult`（第 1218 行起），`deskPnl = quote - payoff`。

## 附录 A：真实数据丰富层（已实现，2026-05-31）

### A.1 设计决策：为什么是「叠加真实参照」而不是「替换教学路径」

考察过同组真实数据后发现：真实危机的崩跌幅度远大于游戏的教学波动带。
- 教学路径刻意用 ±2~3% 的温和波动，好让「敲出 / MCE」的戏剧性恰好落在障碍线上；
- 真实新冠股灾单周可跌 −14%（2020-03 恒指从 24,000 一周内砸到 21,700）。

若直接把真实价格塞进游戏路径，会**击穿精心调好的障碍/收回剧情**，游戏机制反而变得不顺。
因此采用**双层设计**：
- **教学路径**（保留）：仍是确定性的游戏驱动，保证敲出/MCE 剧情干净利落；
- **真实数据层**（新增）：真实日期、真实恒指点位、真实 VHSI，作为「现实参照」卡片显示在路径面板里，把每个虚构剧情**锚定到它对应的真实危机**，并署名数据来源。

这样既用真实数据丰富了内容（拿 Application 分），又给同组数学引擎队的抓数工作做了显式背书，且不破坏游戏手感。
